# CSLINK 管理者マニュアル

CSLINK の運営管理画面を使うための実務ガイド。
新しく管理者になった方は **このドキュメント1本で運用開始** できます。

---

## 1. はじめに

CSLINK には **公開サイト**（cslink.link）と **管理画面**（cslink_admin.html）があります。
このマニュアルは「管理画面で日々何をするか」だけにフォーカスします。
SQL や Supabase 直接操作などの技術手順は [OPERATIONS.md](OPERATIONS.md) を参照してください。

### 管理者ができること
- 企業 / CSプロ の登録内容を一覧で見る
- それぞれの **ステータスを進める**（未確認 → 確認中 → マッチング済 など）
- 企業とCSプロを **マッチング** させる
- マッチング上の **メッセージ** を確認する

---

## 2. ログイン

### URL
**https://cslink.link/cslink_admin.html**

### 認証方式
**マジックリンク**（パスワードレス）。メールに届くリンクを1回クリックするだけ。

### 手順
1. 上記URLを開く
2. メールアドレス欄にあなたの管理者メアドを入力
3. 「ログインリンクを送信」を押す
4. 数十秒以内に **件名「Magic Link for CSLINK」** のメールが届く
5. メール内の **「Log In」ボタンをクリック**
6. 自動でブラウザが管理画面に遷移してログイン完了

### 初めての方
管理者として登録されていないメアドでは「**このアカウントには管理者権限がありません**」と出ます。
川崎（または既存管理者）に依頼して `admin_users` テーブルに追加してもらってください。
追加方法は [OPERATIONS.md「新しい管理者を追加する」](OPERATIONS.md) 参照。

### よくあるエラー
| 表示 | 意味 | 対処 |
|---|---|---|
| `otp_expired` | リンクが古い（5分以上前のもの or 一度使った） | もう一度マジックリンクを送信し、**最新の**メールのリンクを使う |
| `email rate limit exceeded` | 1時間に4通の送信上限に達した | 1時間待つか、別の管理者に頼む |
| `権限がありません` | `admin_users` テーブルに登録されていない | 川崎に登録依頼 |

## 3. 管理画面の構成

ログイン後、画面は **4つのKPIカード** と **3つのタブ** で構成されています。

### 上部：KPIカード（自動更新）
| カード | 表示内容 |
|---|---|
| 登録企業数 | 全企業数 / うち「確認待ち」件数 |
| 登録CSプロ数 | 全CSプロ数 / うち「審査待ち」件数 |
| マッチング数 | 全件数 / うち「成約」件数 |
| メッセージ数 | 全件数 / うち「未読」件数 |

「最終更新」時刻は画面右上に表示。古ければページをリロード。

### 3つのタブ
1. **企業一覧**（タブ初期表示）
2. **CSプロ一覧**
3. **マッチング管理**

---

## 4. 企業一覧タブ

企業（CS課題を抱えてサービス利用したい側）の管理。

### 表示項目
| 列 | 説明 |
|---|---|
| 会社名 | 登録時の会社名 |
| 業種 | プルダウン選択値 |
| CS課題 | 登録時に入力した課題テキスト |
| 担当者 | 担当者名 |
| メール | 連絡先メアド |
| ステータス | 下記参照（カラーバッジ） |
| 登録日 | 登録された日付 |
| 操作 | ステータス変更ドロップダウン |

### ステータスの意味と進め方

```
未確認 → 確認中 → マッチング済 → クローズ
```

| ステータス | 意味 | 次のアクション |
|---|---|---|
| **未確認**（pending / グレー） | 登録されたばかり、運営側未対応 | 内容を確認して「確認中」に進める |
| **確認中**（reviewing / 黄） | 詳細ヒアリング中、または検討中 | マッチング先を選定 |
| **マッチング済**（matched / 青） | CSプロとマッチング作成済み | 進捗を見守る |
| **クローズ**（closed / グレー） | 案件終了（成約 or キャンセル） | アーカイブ扱い |

### ステータスを変更する
1. 該当行の **「操作」列のドロップダウン** を開く
2. 進めたいステータスを選択
3. 自動保存（数秒で「最終更新」時刻が変わる）

---

## 5. CSプロ一覧タブ

CSプロ（CS支援を提供したい個人 / 法人）の管理。**運営側の審査** がここで行われる重要なタブです。

### 表示項目
- 氏名 / 活動形態 / 専門領域 / 経験年数 / 希望単価 / ステータス / 登録日

### ステータスの意味と審査フロー

```
審査待ち → 審査中 → 承認済（公開）
              ↓
              拒否
```

| ステータス | 意味 | 公開サイトでの扱い |
|---|---|---|
| **審査待ち**（pending） | 登録ほやほや、未審査 | 非公開 |
| **審査中**（reviewing） | 運営が経歴・スキルをチェック中 | 非公開 |
| **承認済**（approved） | 審査通過、CSプロ一覧に **公開掲載** | 一覧・詳細ページ・tickerで表示 |
| **拒否**（rejected） | 審査不合格 | 非公開 |

> **重要**: `approved` にすると即時 https://cslink.link/cslink_list_v2.html に表示されます。連絡や面談が済んでから承認してください。

### 審査の流れ（推奨）
1. 新規プロが登録 → ステータス「**審査待ち**」になる
2. 運営側で **審査中** に変更（社内に「対応者」を明示する役割）
3. メアドにメールしてヒアリング日程を設定
4. ヒアリング・スキル確認・経歴の裏取り
5. 問題なければ **承認済** に変更（公開）／問題あれば **拒否**

## 6. マッチング管理タブ

「テスト株式会社が、田中美咲さんに依頼したい」というような関係性を作るタブ。

### マッチングの作成
1. 「マッチング管理」タブ → **「新規マッチング作成」ボタン**
2. ドロップダウンで **企業** と **CSプロ** を選択
3. （任意）マッチ度スコア・メモを入力
4. 「作成」ボタン

→ マッチングが作成され、**両者のマイページ** にカードが表示されます。
→ 同時に **メッセージスレッド** が自動作成され、運営・企業・CSプロの3者でやり取り可能になります。

### マッチングのステータス

```
提案中 → 承諾 → 成約 → クローズ
        ↓
        辞退
```

| ステータス | 意味 |
|---|---|
| **提案中**（proposed） | 運営が提案、両者の返事待ち |
| **承諾**（accepted） | 双方OK、契約調整中 |
| **成約**（contracted） | 契約締結、稼働開始 |
| **辞退**（declined） | どちらかが辞退、または運営判断 |
| **クローズ**（closed） | 案件終了 |

### 成約処理
- マッチングを **「成約」** にすると、KPIカードの「成約数」がカウントアップ
- 必要に応じて **企業のステータスも「クローズ」** に手動で変更（自動連動はしません）

---

## 7. メッセージ確認

各マッチングには **メッセージスレッド** が紐づきます。
- 管理画面のマッチング行を **クリック**（または「詳細」リンク） → メッセージ画面に遷移
- 運営として送信した発言は **「admin」** ラベルで表示される
- 企業側・CSプロ側の発言も時系列で見える

### よくある運営発言例
- 案件マッチングの初回挨拶
- 日程調整の仲介
- 契約条件の確認
- 不適切発言への注意

## 8. ログアウト

画面右上の **「ログアウト」** ボタン。
共有PCを使っている場合は必ずログアウトしてください。

## 9. トラブルシューティング（簡易版）

### 「権限がありません」と出る
→ あなたのメアドが `admin_users` 未登録。川崎に依頼。

### マジックリンクメールが届かない
→ 迷惑メールフォルダを確認。それでもなければ1時間置いて再送（送信上限4通/時間）。

### 古いログインリンクをクリックしたら `otp_expired`
→ メールを再送して **最新の** リンクを使う。

### 一覧に何も出ない
→ そもそもデータが0件。本格運用前のフェーズ。
→ 0件なのに数字が古い場合はページをリロード。

### ステータス変更しても反映されない
→ ネットワーク切れの可能性。**右上「最終更新」時刻** を確認。古ければリロード。

### 詳しいエラー対処は
→ [TROUBLESHOOTING.md](TROUBLESHOOTING.md) に網羅。
